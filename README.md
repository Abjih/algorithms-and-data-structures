# Лабораторные работы по структурам данных и алгоритмам

Репозиторий содержит реализацию классических структур данных и алгоритмов на C++ с полным покрытием unit-тестами (Google Test). Каждая лабораторная работа — самостоятельный модуль с единой структурой проекта.

## Структура репозитория

```
mp2-lab1-bitfield/
mp2-lab2-matrix/
mp2-lab3-stack_rpn/
mp2-lab4-polynomial/
mp2-lab5-tables/
```

Каждая папка содержит:
- `include/` — заголовочные файлы с реализацией
- `gtest/` — фреймворк Google Test
- `test/` — unit-тесты
- `samples/` — примеры использования
- `sln/` — решение Visual Studio

## Лабораторная работа 1: Битовое поле

Реализация класса `TBitField` для хранения и манипуляции битовыми массивами произвольной длины.

**Основные операции:**
- Установка, сброс и чтение отдельных битов
- Битовые операции: AND, OR, NOT, сравнение
- Ввод/вывод через потоки

**Ключевые особенности:**
- Хранение битов в массиве `unsigned int` с побитовой адресацией
- Экономия памяти: один элемент массива хранит 32 бита

## Лабораторная работа 2: Матрицы и специальные форматы хранения

### Динамическая матрица

Шаблонный класс `TDynamicMatrix<T>` на основе `TDynamicVector<T>` с поддержкой:
- Операций с векторами и скалярами (+, -, *, скалярное произведение)
- Матричных операций: сложение, вычитание, умножение матриц, умножение матрица-вектор
- Move-семантика и RAII

### Специальные форматы разреженных матриц

| Формат | Класс | Описание |
|--------|-------|----------|
| **COO** (Coordinate) | `CRD<T>` | Хранение ненулевых элементов в виде тройки (значение, строка, столбец) |
| **CSR** (Compressed Sparse Row) | `CSR<T>` | Хранение значений и столбцов + массив указателей на начало строк |
| **Ленточная** | `BandMatrix<T>` | Хранение диагоналей в пределах полосы ширины `2×band_width+1` |

Все форматы поддерживают конвертацию в плотную матрицу и обратно, а также умножение матриц.

## Лабораторная работа 3: Стек и вычисление выражений

### Стек

Шаблонный класс `Stack<T>` на динамическом массиве с контролем переполнения.

### Парсер в ОПЗ

Класс `Parser` — преобразование инфиксной записи в обратную польскую нотацию (RPN):
- Поддержка операций: `+`, `-`, `*`, `/`
- Поддержка функций: `sin`, `cos`, `ln`, `exp`
- Контроль баланса скобок и приоритетов операторов

### Калькулятор

Класс `Calculator` — вычисление значения выражения в RPN с использованием стека.

## Лабораторная работа 4: Полиномы

Реализация класса `Polynomial` для работы с многочленами от трёх переменных (x, y, z) с однозначными степенями (0–9).

**Структура данных:** односвязный список мономов, упорядоченных по убыванию степени.

**Операции:**
- Сложение, вычитание, умножение полиномов
- Автоматическое приведение подобных членов и удаление нулевых коэффициентов
- Парсинг полинома из строки (поддержка записи вида `3x^2y^1z^0`, `(xyz)^3`, `x*y*z`)

## Лабораторная работа 5: Ассоциативные контейнеры

Реализация 6 структур данных для хранения пар «ключ-значение» (ключ — строка, значение — полином):

| Структура | Класс | Сложность поиска | Особенности |
|-----------|-------|------------------|-------------|
| **Неупорядоченная таблица на массиве** | `UnorderedArrayTable` | O(n) | Линейный поиск, удаление через swap-with-last |
| **Упорядоченная таблица на массиве** | `OrderedArrayTable` | O(log n) | Бинарный поиск, вставка со сдвигом |
| **Упорядоченная таблица на списке** | `OrderedListTable` | O(n) | Dummy-узел, упорядоченность по ключу |
| **Хеш-таблица (метод цепочек)** | `HashTableChains` | O(1) среднее | Разрешение коллизий списками |
| **Хеш-таблица (двойное хеширование)** | `HashTableDouble` | O(1) среднее | Открытая адресация, рехеширование при load factor > 0.5 |
| **Красно-чёрное дерево** | `RBTree` | O(log n) | Самобалансирующееся BST, вставка с перебалансировкой |
| **Скип-лист** | `SkipList` | O(log n) среднее | Вероятностная структура, уровни через coin flip |

## Технологии и практики

- **C++17** — шаблоны, move-семантика, `std::hash`
- **Google Test** — полное покрытие unit-тестами
- **RAII** — управление памятью без утечек
- **ООП** — инкапсуляция, наследование, полиморфизм
- **Алгоритмическая сложность** — осознанный выбор структур под разные паттерны доступа
